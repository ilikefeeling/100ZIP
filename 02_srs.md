## 1. 문서 개요

### 1.1 목적
본 문서는 임대인·임차인 간 입주 정보 인수인계, 계약 금액 상호 확인, 퇴거 정산을 지원하는 임대차 관리 서비스 "건물주"의 기능 및 비기능 요구사항을 정의한다.

### 1.2 범위
- 서비스명: 건물주
- 포함 범위: 모바일 앱(iOS/Android) 및 모바일 웹뷰(임차인 초대 진입 경로)
- 제외 범위: 관리자 백오피스 상세 기능은 최소 범위만 포함하며, 상세 운영 툴은 별도 문서로 관리한다.

### 1.3 용어 정의
| 용어 | 정의 |
|------|------|
| 주거래 중개사 | 임대인에게 앱을 추천·배포하고, 건물의 만기 및 급매물을 전속으로 관리하는 공인중개사 |
| 임대인 | 물건을 등록하고 임차인에게 임대하는 사용자, 다수가 노년층 |
| 임차인 | 초대링크를 통해 계약에 연결되는 입주자 |
| 입주키트 | 임대인이 등록하는 출입정보·설비정보·특약사항 등 생활 인수인계 정보 묶음 |
| 보조 관리자 | 임대인을 대신해 정보를 입력·확인할 수 있는 가족(자녀 등) 계정 |
| 보조 열람자 | 임차인의 정보 열람을 돕는 동거 가족 계정 |
| 더블록(Double-Lock) | 임대인·임차인 양측의 상호 확인·서명이 완료되어야 확정되는 동의 구조 |

### 1.4 참조 문서
- 임대인·임차인 상호 수락 기반 스마트 합의 및 분쟁 제로 UX 전략 (원본 기획서)
- 01_ia.md (건물주 정보구조도)

## 2. 시스템 개요

### 2.1 제품 관점
건물주는 독립형 신규 서비스이며, 카카오/PASS 간편인증과 전자서명 연동 등 외부 인증·서명 시스템과 연계한다. 임차인은 임대인이 발급한 초대링크를 통해 앱 설치 없이 웹뷰로 최초 진입할 수 있는 구조를 갖는다.

**시스템 컨텍스트**
```
[중개사 웹 백오피스] --- 초대링크 발송/고객관리 ---> [건물주 서버]
[임대인 앱] --- 계약등록/입주키트 ---> [건물주 서버] <--- 계약승인/서명 --- [임차인 웹뷰/앱]
[건물주 서버] <---> [PASS/카카오 간편인증]
[건물주 서버] <---> [전자서명 연동 모듈]
[건물주 서버] <---> [알림 발송(SMS/카카오톡/푸시)]
```

### 2.2 사용자 클래스 및 특성
| 사용자 유형 | 설명 | 기술 수준 | 주요 목적 |
|------------|------|-----------|-----------|
| 주거래 중개사 (대표) | 중개사무소 대표(개업공인중개사), 사업자 인증 주체 | 중간 | 직원 관리, 매물 선점, 전체 VIP 임대인 관리 |
| 주거래 중개사 (직원) | 소속공인중개사, 중개보조원 (실장님 등) | 중간 | 배정된 임대인 연락, 실무 및 매물 현장 관리 |
| 임대인 | 물건 소유·임대 관리자, 노년층 비중 높음 | 낮음 | 정보 입력 최소화, 관리 부담 축소 |
| 보조 관리자 | 임대인의 자녀 등 가족 | 중간~높음 | 임대인 대신 입력/확인 대행 |
| 임차인 | 입주자, 상대적으로 젊은 층 다수 | 중간~높음 | 입주정보 즉시 확인, 분쟁 없는 정산 |
| 보조 열람자 | 임차인 동거 가족 | 중간 | 계약/정산 정보 공동 확인 |
| 관리자 | 내부 운영자 | 높음 | 서비스 운영, 분쟁 모니터링 |

### 2.3 운영 환경
- 플랫폼: iOS 15+, Android 10+, 모바일 웹(Chrome/Safari 최신 버전)
- 서버 환경: 클라우드 기반(추후 확정), REST API 서버
- 네트워크: WiFi 및 LTE/5G 환경 모두 지원, 저사양 단말 대응 필요(노년층 사용 단말 특성 고려)

### 2.4 제약사항
- 법적/규제: 개인정보보호법(비밀번호·연락처 등 민감정보 암호화 저장 필수), 전자서명법(공동인증서 또는 간편인증 기반 서명만 법적 효력 인정)
- 기술적: 자유 텍스트 입력을 최소화하는 UI 제약으로 인해 데이터 모델이 선택지 기반 코드값 위주로 설계되어야 함
- 사업적: MVP는 입주키트+전자서명 계약등록까지, 2단계 금액확인, 3단계 퇴거정산 순으로 단계적 출시

## 6. 제약사항 및 가정
- 임대인의 상당수가 스마트폰 조작에 익숙하지 않다고 가정하며, 모든 신규 화면 설계 시 자유 텍스트 입력 최소화 원칙을 우선 적용한다.
- 임차인의 초대링크 진입은 앱 미설치 상태에서도 웹뷰로 핵심 기능(계약확인, 입주키트 열람, 서명)이 100% 동작해야 한다.
- 관리비는 정액(1식) 부과를 기본 가정하며, 별도 관리비 정산 기능은 범위에서 제외한다.
- 2단계 금액 확인 기능은 협상(카운터오퍼) 로직을 포함하지 않는다.
- 임차료 납부 관리(RENT)는 계약 생애주기와 별개로 매월 반복되는 상시 기능이며, 3단계(퇴거정산)와 병렬로 상시 운영된다. 이 기능이 서비스의 월간 재사용(리텐션) 핵심 축이다.
- 임차료 납부 관리는 1차 범위에서 실제 PG 결제 대행을 포함하지 않으며, 납부 안내·확인·독촉과 미납/연체 기록 관리에 한정한다. 실제 계좌이체 확인은 임대인의 수동 확인 또는 (2차 범위) 계좌 연동 자동확인으로 처리한다.
- 세무 신고자료 생성 기능은 정식 세무 서식(종합소득세 임대소득 신고 등) 요건에 대한 세무 전문가 검토가 별도로 필요하며, 본 문서에서는 "기간별 임대소득 현황 요약 자료 생성"까지를 1차 범위로 한다.
- 시세 참고자료 기능은 이번 범위에서 메뉴 진입점만 구현하고 "준비중" 상태로 노출한다. 실제 계산 로직은 2차 고도화 범위이다.
- 주거래 중개사 플랫폼은 PC 모니터 화면 비율을 고려한 반응형 '모바일/PC 웹 백오피스' 형태로 별도 개발되며, 앱에는 포함되지 않는다.
- 중개사 가입 시 사업자등록번호 인증은 무료인 '공공데이터포털 국세청 사업자등록상태조회 API'를 우선 연동하여 비용 발생을 차단한다.
