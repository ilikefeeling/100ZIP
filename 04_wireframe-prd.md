## 0. 디자인 시스템 기본값 (전 화면 공통 준수)

| 항목 | 값 | 사유 |
|------|-----|------|
| 기본 본문 글자 크기 | 18pt 이상 | 노년층 가독성 |
| 강조/제목 글자 크기 | 24~28pt | 화면당 핵심 질문 1개 강조 |
| 버튼 최소 터치 영역 | 48x48dp | NFR-UX-001 |
| 버튼 톤 | 실행(초록)/취소(회색)/경고(빨강) 3색만 사용 | 색상 인지 부담 최소화 |
| 입력 방식 우선순위 | ① 카드선택 → ② 숫자패드 → ③ 사진촬영 → ④ 텍스트(숨김, 최후수단) | NFR-UX-003 |
| 화면당 질문 수 | 1개 원칙 | 다단계 폼 금지 |
| 하단 고정 영역 | 고객센터 전화 버튼 상시 노출 | COM-006 |
| 뒤로가기/취소 | 모든 입력 화면 상단 좌측 고정 | 조작 실수 방지 |

## 1. 공통/인증 플로우

### COM-001 스플래시
- **레이아웃**: 중앙 로고 + 하단 로딩 인디케이터, 텍스트 없음
- **분기**: 토큰 유효 시 역할별 홈 자동 이동(0.5초 이내), 무효 시 COM-002

### COM-002 역할 선택
- **레이아웃**: 화면 상/하 2분할, 각 영역 큰 버튼 1개
  - 상단: "임대인이에요" (건물 아이콘)
  - 하단: "초대코드로 입장할게요" (열쇠 아이콘)
- **인터랙션**: 버튼 탭 시 즉시 다음 화면, 별도 확인 팝업 없음

### COM-003 본인인증 로그인
- **레이아웃**: 안내 문구 1줄("휴대폰 인증만으로 시작할 수 있어요") + 카카오/PASS 버튼 세로 배치(각 56dp 높이)
- **상태**: 인증 진행 중 전체화면 로딩, 실패 시 화면 유지 + 상단 토스트

### COM-004 최초 이용 안내
- **레이아웃**: 4장 슬라이드 카드(스와이프), 각 카드 큰 일러스트 1개 + 짧은 문장 1줄
  1. "임대 정보를 한 번만 입력하세요"
  2. "세입자에게 자동으로 전달돼요"
  3. "매달 임차료도 알아서 챙겨드려요"
  4. "이제 시작해볼까요?" + [시작하기] 버튼
- **인터랙션**: 언제든 하단 "건너뛰기" 노출

### COM-005 알림센터
- **레이아웃**: 상단 탭 없이 최신순 리스트, 카드형(아이콘+제목+시간)
- **컴포넌트**: 미확인 알림 좌측 점 표시, 탭 시 해당 화면 딥링크 이동

### COM-006 고객센터
- **레이아웃**: 대형 전화 버튼(화면 상단 고정, "전화로 물어보기") + 하단 FAQ 아코디언
- **비고**: 모든 화면 하단 탭바 옆에 미니 버튼으로 상시 접근 가능해야 함(전역 컴포넌트)

### COM-007 초대링크 진입(웹뷰)
- **레이아웃**: 앱 설치 없이 노출되는 첫 화면 — 물건 사진 + 주소 + 보증금/월세 요약 카드
- **CTA**: 하단 고정 [본인인증하고 확인하기] 버튼 1개
- **비고**: 반응형 웹, 모바일 세로 화면 기준 최적화(모바일 브라우저 대응 최우선)

## 2. 임대인 — 건물/호실 등록 및 임차인 변경

### LL-001 임대인 홈(물건 목록)
- **레이아웃**: 상단 "안녕하세요, ○○님" 인사 + 건물 카드 리스트(주소, 공실/임대중 요약 뱃지)
- **CTA**: 우하단 큰 [+ 건물 추가] 플로팅 버튼
- **하단 탭바 (4개, 아이콘+텍스트)**: 홈 / 계약관리 / 납부관리 / 알림

### LL-002 건물 등록
- **레이아웃**: 순차 카드
  1. "주소를 검색해주세요" — 주소 검색 API 리스트 선택
  2. "어떤 건물인가요?" — 카드 선택(원룸/다가구/오피스텔/상가주택)
  3. "총 몇 호실인가요?" — 숫자패드
  4. "사진을 찍어주세요" — 카메라 버튼(건물 외관)
- **완료 후**: 총 호실수만큼 빈 호실 슬롯이 자동 생성되어 LL-002a로 이동

### LL-002a 호실 목록
- **레이아웃**: 건물 상단 요약(주소, 공실/임대중 비율 미니 표시) + 호실 카드 그리드(2열)
- **카드 표시**: 호실번호 + 상태 뱃지(공실=회색/임대중=초록) + 미입력 슬롯은 "정보 입력하기" 흐린 카드로 구분
- **CTA**: 각 미입력 슬롯 탭 시 LL-002b로, 입력완료 호실 탭 시 LL-002c로

### LL-002b 호실 등록
- **레이아웃**: 순차 카드
  1. "호실 번호를 확인해주세요" — 자동제안값(예: "101호") 수정 가능한 숫자+텍스트 프리셋
  2. "전용면적이 어떻게 되나요?" — 숫자패드(㎡)
  3. "구조는요?" — 카드(원룸/투룸/쓰리룸이상)
  4. "어떤 옵션이 있나요?" — 아이콘 그리드 다중선택(에어컨/냉장고/세탁기/붙박이장/가스레인지/엘리베이터)
  5. "특이사항이 있나요?" — 프리셋 카드(반려동물가능/주차가능/없음) + "직접 입력" 숨김 버튼
- **인터랙션**: 완료 시 호실 상태 "공실"로 저장, LL-002a로 복귀

### LL-002c 호실 상세
- **레이아웃**: 상단 호실 기본정보 요약 카드(면적/구조/옵션 아이콘 나열) + 현재 계약정보 카드(공실 시 "공실입니다" 안내) + 하단 메뉴 리스트
- **메뉴 리스트 구성**:
  - [계약 등록하기] (공실 상태일 때만 노출) → LL-003
  - [임차인 변경] (임대중 상태일 때만 노출) → LL-003a
  - [임차인 이력 보기] → 타임라인(과거 계약금/기간/임차인 리스트)
  - [시세 참고자료] (준비중 뱃지) → LL-024
  - [매물카드 자동생성] (준비중 뱃지) → LL-025

### LL-003 계약 등록(금액)
- **레이아웃**: 순차 카드
  1. "세입자 이름을 알려주세요" — 텍스트(짧은 고유명사라 예외적으로 자유입력 허용)
  2. "세입자 연락처는요?" — 숫자패드(전화번호 형식)
  3. "보증금은 얼마인가요?" — 숫자패드
  4. "월세는요?" — 숫자패드 (전세인 경우 0 입력 또는 "전세예요" 토글)
  5. "계약 기간은요?" — 캘린더 2회(시작일/종료일)
  6. "임차료 받는 날은 매월 며칠인가요?" — 날짜 프리셋(1일/5일/10일/말일/직접선택)
- **비고**: 마지막 질문이 임차료 납부관리 스케줄(FR-RENT-001)의 기준값이 되므로 필수 입력으로 처리

### LL-003a 임차인 변경
- **레이아웃**: 단일 확인 화면 — "현재 세입자와의 계약을 종료하고 새 세입자를 등록할까요?" + 정산 상태 카드(완료/미완료 표시)
- **분기**:
  - 정산 미완료 시: [퇴거정산 하러가기] 버튼만 활성화(다음 단계 잠금)
  - 정산 완료 시: [새 세입자 등록하기] 버튼 활성화 → LL-003으로 이동, 입주키트 화면에 "이전과 동일하게 사용" 토글 상단 노출(재입력 부담 축소)

## 3. 임대인 — 공실 현황 및 계약갱신 관리

### LL-022 공실 현황 대시보드
- **레이아웃**: 건물별 카드, 각 카드에 공실률 도넛형 미니 그래프(간단한 %숫자 강조) + "공실 3개월 이상: 202호" 식의 최장 공실 호실 하이라이트 1줄
- **비고**: 복잡한 차트 대신 큰 숫자 중심 — 노년층 임대인이 한눈에 "몇 개 비어있다"만 파악하면 되는 수준으로 단순화

### LL-023 계약갱신 관리
- **레이아웃**: 계약만료 임박 호실 카드 리스트, D-day 뱃지(D-60=회색, D-30=주황) 순 정렬
- **카드 탭 시**: [임차인 변경] 또는 [금액 조정 후 갱신](LL-011 연결) 2개 버튼 제공

### LL-024 시세 참고자료 (준비중)
- **레이아웃**: 중앙 아이콘 + "곧 만나요, 시세 참고자료" 안내 문구 1줄 + [알림 받기] 버튼(향후 오픈 시 알림)

- **비고**: 로직 없음, 메뉴 누락 방지용 자리표시

## 5. 임대인 — 입주키트 입력

공통 패턴: 각 화면은 "카드 스택 + 건너뛰기" 구조. 상단 진행률 바(예: "3/5 항목") 노출.

### LL-004 출입보안정보
- **레이아웃**: 4개 카드 순차 노출, 각 카드는 큰 라벨(예: "공동현관 비밀번호") + 숫자키패드(전화 잠금해제 UI 동일)
- **카드 순서**: 공동현관 → 세대현관 → 우편함 → 주차
- **건너뛰기**: 카드 우상단에 작게 "몰라요" 텍스트 버튼

### LL-005 설비/생활정보
- **레이아웃**: 카드형 선택지(아이콘 그리드), 항목별:
  - 보일러: [경동] [귀뚜라미] [린나이] [사진으로 대체]
  - 인터넷: [SKT] [KT] [LG U+] [모름]
  - 가스: [도시가스] [개별계량] [모름]
- **사진으로 대체 선택 시**: 카메라 화면 즉시 전환

### LL-006 관리비 항목
- **레이아웃**: 큰 카메라 버튼 중앙 배치("관리비 고지서를 찍어주세요") → 촬영 후 OCR 인식 결과를 표 형태로 자동 표시 → "맞아요"/"직접 수정" 2버튼
- **직접 수정 시**: 항목별 숫자패드만 노출(텍스트 라벨은 고정 프리셋: 일반관리비/청소비/승강기/공용전기/인터넷)

### LL-007 특약사항
- **레이아웃**: 카드형 2지선다 반복
  - "반려동물 가능한가요?" [가능]/[불가능]
  - "실내 흡연 가능한가요?" [가능]/[불가능]
  - "원상복구 범위는요?" [기본(못자국 등 통상마모 제외)]/[직접 정하기]
  - "장기수선충당금은 누가 돌려받나요?" [임대인]/[임차인 정산]
- **직접 정하기 선택 시에만** 자유 텍스트 입력창 노출

### LL-008 비상연락처
- **레이아웃**: 3개 카드(임대인 본인/관리업체/설비기사), 각 카드는 전화번호 숫자패드 + "통화 테스트" 버튼

### LL-009 초대링크 발급
- **레이아웃**: "세입자 번호를 입력해주세요" 숫자패드 → 발급 완료 화면에서 [문자로 보내기]/[카카오로 보내기] 큰 버튼 2개
- **상태 표시**: 발급 후 "세입자가 아직 확인 전이에요" 대기 상태 카드, 만료일 D-day 카운트

## 6. 임차인 — 계약 확인 및 입주키트

### TN-001 임차인 홈
- **레이아웃**: 계약 요약 카드(주소, 임대인명, 계약상태) + 입주키트 진행률 바 + 이번달 납부 상태 위젯
- **하단 탭바**: 홈 / 우리집정보(입주키트) / 납부내역 / 알림

### TN-002 계약정보 확인/승인
- **레이아웃**: 계약 요약서 전체 스크롤(보증금/월세/기간/특약) + 하단 고정 [확인했습니다] 버튼
- **비고**: 스크롤 끝까지 확인해야 버튼 활성화(중요 정보 누락 확인 방지)

### TN-003 입주키트 열람
- **레이아웃**: 탭 구조(출입정보/설비정보/특약사항/비상연락처), 각 탭은 아이콘 카드 리스트
- **미입력 항목**: 흐리게 표시 + "임대인에게 물어보기" 버튼 인라인 노출(TN-004 연결)

### TN-005 입주 하자 스탬프 등록

  - "미납 독촉은 며칠 후에 보낼까요?" — 프리셋(3일후/7일후/10일후)

- **정렬**: 미납일수 긴 순으로 기본 정렬

### LL-020 세무 신고자료 생성 (기초 버전)

- **제외(2차 고도화)**: 정식 세무서식 매핑, 물건별 경비 자동 반영, 세무사 연동 등은 이번 범위에서 제외
- **레이아웃**: "올해"/"작년" 큰 버튼 2개 → 생성 중 로딩 → 완료 시 다운로드 버튼 2개(PDF/CSV) + 상단 고지문("참고용 자료입니다") 고정 노출

### TN-009 임차료 납부 안내
