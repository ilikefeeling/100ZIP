# AGENTS.md — 건물주 프로젝트 에이전트 작업 지침

## 프로젝트 개요
"건물주"는 노년층 임대인을 주 사용자로 하는 임대차 관리 앱(웹뷰+앱 병행)입니다.
핵심 가치: 임대인은 관리가 쉽고, 임차인은 입주 첫날부터 헤매지 않는 것.

## 작업 시작 전 필수 확인 문서
아래 문서를 **순서대로** 먼저 읽고 작업을 시작하세요. 구현 중 애매한 부분이 있으면 임의로 추측하지 말고 아래 문서를 다시 확인하세요.

1. `01_ia.md` — 전체 화면 구조(IA), 화면ID 체계, 화면 간 이동 관계
2. `02_srs.md` — 기능 요구사항 전체(FR-ID 체계), 우선순위([P1]/[P2]/[P3]), 비기능 요구사항
3. `03_functional-spec.md` — 핵심 기능별 상세 로직, API 스펙, 에러 코드, 예외처리
4. `04_wireframe-prd.md` — 화면별 레이아웃, 컴포넌트 구성, 인터랙션 상세
5. `05_design-tokens.md` — 컬러/폰트/간격 CSS 변수 토큰 (구현 시 반드시 이 값을 그대로 사용)

> 위 6개 파일(AGENTS.md 포함)은 모두 프로젝트 루트에 같은 위치로 둡니다. 별도 `docs/` 하위 폴더로 옮길 필요 없습니다.

## 절대 원칙 (코드 구현 시 반드시 준수)

### 1. 노년층 임대인 UX 원칙
- 자유 텍스트 입력 **최소화**. 카드 선택형 → 숫자패드 → 사진촬영 순으로 우선 사용
- 화면당 질문(입력 요구) 1개 원칙, 다단계 폼 금지
- 모든 터치 영역 최소 48x48dp, 기본 버튼 높이 56px, 숫자패드 키 최소 60px
- 본문 글자 최소 18px(=`--font-body` 토큰), 그 이하 크기는 캡션 용도 외 사용 금지
- 모든 입력 화면에 "건너뛰기"/취소 옵션 상시 제공

### 2. 디자인 토큰
- `docs/05_design-tokens.md`의 CSS 변수를 그대로 프로젝트에 이식(`styles/tokens.css` 등으로 분리 권장)
- 임의로 색상·폰트 크기를 새로 정의하지 말고 반드시 토큰 참조
- 상태 표시(납부완료/미납/독촉중 등)는 색상만이 아니라 텍스트 라벨을 항상 병기

### 3. 데이터 계층
건물(Building) → 호실(Unit, N개) → 계약(Contract) → 임차인(Tenant) 4단계 계층을 기준으로 데이터 모델을 설계하세요. 단일 세대 구조로 단순화하지 마세요(다가구/다호실 건물주가 주 타겟입니다).

### 4. MVP 범위 (이번 스프린트에서 구현할 것)
- 인증(간편인증 로그인), 건물/호실 등록, 입주키트(출입보안/설비/관리비/특약/비상연락처)
- 초대링크 발급 및 임차인 웹뷰 진입
- 전자서명 계약 확정
- 금액 상호확인(협상 UI 없음, 단순 승인)
- 퇴거정산 계산기
- 임차인 변경(정산 완료 강제 확인 후 진행)
- 임차료 납부관리(안내→확인→미납→독촉 4단계 알림, 미납자 대장)
- 공실 현황 대시보드, 계약갱신 임박 알림(D-60/D-30)
- 세무 신고자료는 **기초 버전만**(연간 합계표 생성+다운로드, 정식 서식화 아님)

### 5. 이번 범위에서 제외(스텁만 구현)
- 시세 참고자료 메뉴 (`01_ia.md`의 LL-024) — "준비중" 화면만, 로직 없음
- 매물카드 자동생성 메뉴 (LL-025) — "준비중" 화면만, 로직 없음
- 보조 관리자/보조 열람자 초대, 불편사항 전달 — 이번 스프린트 제외

## 작업 진행 방식
- 한 번에 전체를 구현하려 하지 말고, `02_srs.md`의 우선순위([P1]부터) 순서로 기능 단위로 작업하고 각 단위마다 결과를 보고하세요.
- 화면 구현 시 반드시 `04_wireframe-prd.md`의 해당 화면ID 섹션을 먼저 확인하고, 레이아웃 임의 변경 금지(변경이 필요하면 먼저 이유를 설명하고 확인받으세요).
- API 설계 시 `03_functional-spec.md`의 엔드포인트 명세와 공통 에러 코드 체계를 그대로 따르세요.

## 기술 스택 (미확정 시 제안)
- 프론트: React 기반 웹뷰 + 앱(React Native 또는 하이브리드 검토)
- 인증: PASS/카카오 간편인증 연동
- 전자서명: 간편인증 기반 재인증 방식(서명패드 없음)
